Flujo paso a paso
1
Raspberry Pi · camera.py
La cámara detecta una cara
OpenCV captura el frame. Si detecta un rostro humano, lo recorta y lo pasa al módulo de codificación.
2
Raspberry Pi · encoder.py
Convierte la cara en un código numérico
La librería face_recognition convierte la imagen del rostro en 128 números (vector biométrico). Ese vector es único por persona.
3
Raspberry Pi · sender.py
Envía el vector al servidor por WiFi
Hace una petición HTTP POST a /api/facial con el vector, el ID del dispositivo y la fecha/hora. Si falla, guarda el evento en eventos_locales.log y reintenta.
4
API Gateway · auth.middleware.js
El portero verifica que el dispositivo está autorizado
Valida el token JWT del dispositivo Raspberry. Si no tiene token válido, rechaza la petición con 401. Si pasa, la reenvía al servicio biométrico.
5
BiometricService · matcher.js
Compara el vector con los rostros registrados
Busca en la tabla RostroUsuario todos los vectores activos y calcula la distancia euclidiana. Si la distancia es menor a 0.6, el usuario es reconocido. Guarda un EventoFacial en la base de datos.
6
Event Bus · eventBus.js
Se emite el evento "facial_detected"
El BiometricService publica el evento con los datos del usuario, ambiente y timestamp. Varios servicios escuchan este evento al mismo tiempo.
7
AttendanceService · validacion.js
Valida si el aprendiz está en el ambiente correcto
Consulta el horario activo del aprendiz. Si el ambiente coincide → registra entrada/salida. Si no coincide → registra anomalía (ambienteIncorrecto). También calcula si hay retraso.
8
Event Bus
Se emite "attendance_registered" o "anomaly_found"
Dependiendo del resultado anterior, se lanza el evento correspondiente. El NotificationService escucha ambos y reacciona automáticamente.
9
NotificationService · generator.js
Genera notificación y envía correo si aplica
Crea un registro en la tabla Notificacion. Si el canal es EMAIL, usa el SMTP para enviar correo al instructor o acudiente. Guarda el resultado en EnvioCorreo.
10
Frontend · websocket.js
El dashboard se actualiza en tiempo real
El servidor envía la notificación por WebSocket. El Redux notifSlice actualiza el estado. El aprendiz ve su asistencia registrada y el contador de notificaciones sube.
11
BitacoraSistema · logger.js
Cada paso queda registrado en la bitácora
En cada paso del flujo se guarda: módulo, usuario, IP de origen, acción, resultado y fechaHora. Esto cumple los requerimientos RF-11 de auditoría.
