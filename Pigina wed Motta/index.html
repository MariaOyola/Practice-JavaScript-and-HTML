Introducción

Introducción

El SENA ha tenido problemas por el retraso de los aprendices y acumulación de fallas de manera constante, lo que involucra un registro manual obligatorio que deben realizar los instructores. Lo que conlleva a una pérdida de tiempo y mal control, pues los instructores deben llenar la lista por cada uno de los aprendices, preguntar si está o no y anotarlo. Todos los días se repite el mismo proceso; en ocasiones hasta se olvida llamar a asistencia.

Propósito

El propósito del presente sistema es diseñar e implementar una solución tecnológica que permita automatizar el proceso de control y gestión de asistencia mediante reconocimiento facial, reemplazando el registro manual realizado actualmente por los instructores en hojas de cálculo (Excel).

El sistema busca garantizar un registro preciso, automático y en tiempo real de la asistencia de los aprendices dentro del ambiente de formación, permitiendo identificar de manera confiable:

✅ Quién asistió
🕐 Quién llegó puntual
⚠️ Quién llegó tarde
❌ Quién no asistió

De esta manera, se pretende reducir los errores humanos, evitar omisiones involuntarias al momento de llamar lista y optimizar el tiempo operativo del instructor, mejorando la eficiencia en la gestión académica. Además, el sistema permitirá generar reportes organizados y actualizados que faciliten el seguimiento del comportamiento de asistencia de cada aprendiz, apoyando la toma de decisiones académicas y administrativas.

Alcance

Este sistema de reconocimiento facial para gestión de asistencia tendrá el siguiente alcance:

👤 Registro de Usuarios

  • Permitir el registro previo de los aprendices.
  • Almacenar datos básicos (nombre, ficha, programa).
  • Capturar y guardar información biométrica facial para reconocimiento.

📷 Reconocimiento Facial en Tiempo Real

  • Detectar automáticamente el rostro del aprendiz al ingresar al ambiente.
  • Comparar el rostro con la base de datos registrada.
  • Identificar al aprendiz sin intervención manual del instructor.

📝 Registro Automático de Asistencia

  • Registrar automáticamente la hora exacta de ingreso.
  • Clasificar la asistencia: presente puntual, presente con retraso, o ausente.

📊 Gestión y Consulta de Reportes

  • Generar reportes individuales y grupales.
  • Visualizar historial de asistencia y estadísticas.
  • Exportar reportes en formato Excel o PDF.

Descripción General del Sistema

🎯 Objetivo del sistema

El sistema tiene como objetivo automatizar el control de asistencia académica mediante reconocimiento facial para mejorar la precisión, seguridad y eficiencia del registro de asistencia de los aprendices. De esta manera se busca reemplazar el proceso manual que realizan los instructores, reduciendo errores humanos, evitando suplantaciones de identidad y optimizando el tiempo dedicado al registro de asistencia.

Además, el sistema permitirá generar reportes automáticos que faciliten el seguimiento del comportamiento de asistencia de los estudiantes.

👥 Usuarios del sistema

🛡️ Administrador

Responsable de gestionar el funcionamiento general del sistema, administrar usuarios y supervisar la información de la plataforma.

👨‍🏫 Instructor

Puede registrar aprendices, asociarlos a fichas y consultar los reportes de asistencia generados por el sistema.

🎓 Aprendiz

Identificado por el sistema mediante reconocimiento facial para registrar automáticamente su asistencia al ingresar al ambiente de formación. No requiere intervención manual.

⚡ Funcionalidades principales

El sistema permitirá el inicio de sesión con control de acceso por roles. También registrará aprendices con sus datos personales y almacenará su información facial. Al ingresar al ambiente, el sistema detectará el rostro mediante cámara, lo identificará y registrará automáticamente la asistencia con la hora de ingreso.

Adicionalmente gestionará programas de formación, fichas, ambientes y horarios académicos, y generará reportes exportables en PDF o Excel.

🌐 Contexto del sistema
El sistema será utilizado dentro del entorno académico del SENA. Funcionará mediante una plataforma web conectada con un dispositivo de reconocimiento facial basado en Raspberry Pi y una cámara. El dispositivo capturará los rostros de los aprendices y enviará la información al servidor a través de red WiFi, registrando la asistencia en tiempo real en la base de datos.

Punto 08

Reglas de Negocio

Las reglas de negocio definen las condiciones, restricciones y comportamientos que el sistema debe cumplir obligatoriamente. Están organizadas por área funcional.

🔐 Usuarios y Seguridad 7 reglas
RN-01

Un usuario no puede registrarse con un correo electrónico que ya exista en el sistema.

RN-02

Cada usuario debe tener un único perfil de credenciales activo.

RN-03

Solo los usuarios mayores de edad pueden registrarse como adultos; los menores requieren consentimiento de un acudiente.

RN-04

Cada usuario puede tener uno o varios roles, pero un rol solo se asigna una vez por usuario.

RN-05

Las contraseñas deben cumplir la política de seguridad definida: longitud mínima, mayúsculas, números y símbolos.

RN-06

Después de tres intentos fallidos de inicio de sesión, la cuenta se bloquea temporalmente.

RN-07

Los administradores pueden crear, modificar o eliminar usuarios y asignar roles y permisos.

📷 Asistencia y Reconocimiento Facial 6 reglas
RN-08

Cada asistente debe ser identificado por reconocimiento facial antes de registrar su asistencia.

RN-09

Un usuario solo puede registrar una entrada y una salida por horario asignado.

RN-10

Si un usuario llega fuera del horario definido, se marcará como "tardío".

RN-11

La asistencia se registra automáticamente en el sistema con fecha, hora y ambiente.

RN-12

No se permite la suplantación de identidad; cada rostro corresponde a un único usuario registrado.

RN-13

Los eventos de reconocimiento facial deben enviarse al servidor para validación antes de actualizar la asistencia.

🏛️ Gestión de Ambientes y Horarios 4 reglas
RN-14

Un ambiente solo puede estar asignado a una ficha en un mismo horario.

RN-15

Si un ambiente está marcado como restringido o cerrado, no se puede asignar a ninguna ficha temporalmente.

RN-16

Cada ficha debe tener un horario asignado y un ambiente disponible para registrar asistencia.

RN-17

Los instructores deben estar asociados a los horarios de las fichas que supervisan.

📊 Reportes y Consultas 3 reglas
RN-18

Los reportes de asistencia deben mostrar el estado de puntualidad, inasistencias y ausencias justificadas.

RN-19

Solo los usuarios con permisos adecuados pueden generar y exportar reportes.

RN-20

Las excusas de inasistencia deben aprobarse antes de que se reflejen en los reportes oficiales.

🔔 Notificaciones y Comunicación 2 reglas
RN-21

Las notificaciones deben enviarse automáticamente ante ausencias, anomalías o cambios en horarios y ambientes.

RN-22

Cada usuario recibe notificaciones únicamente sobre sus registros, horarios o eventos relevantes.

⚙️ Configuración y Personalización 2 reglas
RN-23

Cada usuario puede configurar su idioma, colores y modo del dashboard.

RN-24

La personalización es única por usuario y no afecta la configuración de otros usuarios.

📋 Auditoría y Bitácoras 3 reglas
RN-25

Todas las acciones de usuarios y administradores deben registrarse en la bitácora del sistema.

RN-26

Las auditorías y bitácoras no se pueden eliminar; solo pueden ser visualizadas por administradores.

RN-27

Cada error registrado debe incluir información de fecha, usuario y tipo de error para su control.

⚖️ Legal y Consentimiento 3 reglas
RN-28

Todo usuario debe aceptar los términos y condiciones antes de usar el sistema.

RN-29

Los menores de edad requieren consentimiento de un acudiente para registrar su información biométrica.

RN-30

El consentimiento de datos biométricos debe validarse y registrarse para cumplir con las normativas legales.

Seguridad del Sistema

El sistema implementa múltiples capas de seguridad para garantizar la confidencialidad, integridad y disponibilidad de la información. Combina autenticación por credenciales, reconocimiento facial, control de acceso por roles y protección de datos conforme a la Ley 1581 de 2012 de Colombia.

🔑 Autenticación

El sistema combina autenticación por credenciales y autenticación biométrica facial, garantizando mayor seguridad y confiabilidad en la verificación de identidad.

📧 Por credenciales

Acceso mediante correo y contraseña validados contra la base de datos. Las contraseñas se almacenan con hashing seguro, nunca en texto plano.

8–15 caracteres Letras + números + símbolos Max 3 intentos fallidos Registro de auditoría

📷 Biométrica facial

Durante el registro inicial se captura el rostro mediante Raspberry Pi, generando un vector biométrico. Al ingresar, la cámara compara el rostro con los registros almacenados para identificación automática.

Captura con Raspberry Pi Vector biométrico Comparación en tiempo real
Estas medidas garantizan la protección de las credenciales y el cumplimiento de la Ley 1581 de 2012 sobre protección de datos personales en Colombia.
🛡️ Autorización

El sistema implementa autorización basada en roles (RBAC), garantizando que cada usuario solo acceda a las funciones correspondientes a su perfil.

⚠️ Derechos de datos personales

Antes de usar el sistema, el usuario debe confirmar que ha leído y comprendido sus derechos de acceso, actualización y rectificación. La confirmación se registra con fecha, hora y usuario. Si no acepta, el acceso se bloquea.

🛡️ Administrador general

Acceso completo para gestionar usuarios, roles, ambientes, fichas, horarios y configuraciones. Puede suspender cuentas, restablecer contraseñas y consultar reportes globales. Todas sus acciones quedan en bitácora.

🚪 Control de Acceso
📡
Envío seguro desde Raspberry Pi

Cada reconocimiento facial envía un paquete con ID usuario, ambiente, ficha, fecha, hora y tipo de evento. El servidor valida antes de almacenar. Si no hay red, los eventos se sincronizan al restablecer conexión.

📋
Bitácoras del sistema

Se registran todas las acciones relevantes: accesos, movimientos de asistencia, cambios de configuración y actividades de usuarios. Cada registro incluye usuario, acción, fecha, hora e IP.

🔍
Módulo de auditoría

Permite a administradores y auditores consultar reportes sobre accesos, movimientos y anomalías, asegurando control y supervisión completa del uso del sistema.

🔒 Protección de Datos

El sistema garantiza la protección de datos personales y biométricos mediante mecanismos de seguridad en el almacenamiento y transmisión. Los datos capturados por la Raspberry Pi se envían al servidor de forma segura para evitar accesos no autorizados.

🔏 Confidencialidad Solo usuarios autorizados acceden a los datos
✔️ Integridad Los datos no pueden ser alterados sin autorización
🌐 Disponibilidad El sistema y sus datos están siempre accesibles
Cumple con la Ley 1581 de 2012 sobre protección de datos personales en Colombia. Solo usuarios con permisos autorizados pueden acceder o modificar datos.
🔄 Manejo de Contraseñas

El sistema cuenta con un mecanismo seguro de restablecimiento de contraseñas mediante token único y temporal enviado al correo del usuario.

Instalación y Configuración

Procedimientos paso a paso para instalar y configurar cada herramienta del entorno de desarrollo del proyecto.

💻 Visual Studio Code — IDE
Instalación
1
Descargar desde https://code.visualstudio.com → clic en Download for Windows.
2
Ejecutar el instalador (.exe) · Aceptar términos · Next en todo.
!
Marcar: Add to PATH · Create a desktop icon · Open with Code
3
Clic en Install → Finish. VS Code se abre automáticamente.
Configuración inicial
1
Idioma: Extensiones (Ctrl+Shift+X) → buscar Spanish Language Pack → instalar → reiniciar.
2
Tema: Ctrl+K luego Ctrl+T → elegir tema (ej: Dark+).
Configuración para Java
1
Instalar el JDK 17 o 21 (ver sección Java abajo).
2
Configurar JAVA_HOME en variables de entorno del sistema.
3
Verificar en consola: java -version · javac -version
☕ Java — Lenguaje de programación
Instalar JDK
1
Descargar desde Oracle o OpenJDK. URL: https://www.oracle.com/java/technologies/downloads/
2
Ejecutar instalador → Next → Next → Install (ruta por defecto recomendada).
Variables de entorno (Windows)
1
Buscar "Variables de entorno" → Editar variables del sistema.
2
Nueva variable: JAVA_HOME = C:\Program Files\Java\jdk-21
3
Editar Path → agregar %JAVA_HOME%\bin
Verificación y Hola Mundo
1
En CMD: java -version · javac -version → si sale la versión ✓
✓
// HolaMundo.java
public class HolaMundo {
  public static void main(String[] args) {
    System.out.println("Hola mundo");
  }
}
javac HolaMundo.java → java HolaMundo
🐍 Python — Lenguaje de programación
Instalación
1
Descargar desde https://python.org la última versión.
!
Marcar obligatoriamente: Add Python to PATH → Install Now.
2
Verificar: python --version · pip --version
Entorno virtual y librerías
1
Crear entorno: python -m venv mi_entorno
2
Activar (Windows): mi_entorno\Scripts\activate
3
Instalar librerías: pip install numpy · pip install pandas
Hola Mundo
✓
# hola.py
print("Hola mundo")
python hola.py
🍃 Spring Boot — Backend
Creación del proyecto (recomendado)
1
Ir a https://start.spring.io (Spring Initializr).
2
Configurar: Project: Maven · Language: Java · Dependencies: Spring Web, DevTools.
3
Clic en Generate → descargar ZIP → descomprimir → abrir en IDE.
4
Ejecutar: mvn spring-boot:run
⚛️ React 19 + Vite + Tailwind CSS — Frontend
Requisito: Node.js
1
Descargar versión LTS desde https://nodejs.org.
2
Verificar: node -v · npm -v
Crear proyecto React + Vite
1
npm create vite@latest → nombre: mi-app · framework: React · variante: JavaScript
2
cd mi-app → npm install → npm run dev
3
Abrir en navegador: http://localhost:5173
Instalar Tailwind CSS
1
npm install -D tailwindcss postcss autoprefixer
2
npx tailwindcss init -p
3
En tailwind.config.js: agregar "./src/**/*.{js,ts,jsx,tsx}" en content.
4
En src/index.css: @tailwind base; @tailwind components; @tailwind utilities;
🗄️ PostgreSQL — Base de Datos
Instalación
1
Descargar desde https://www.postgresql.org/download/
2
Ejecutar instalador → seguir pasos.
!
Definir contraseña del usuario postgres · Puerto: 5432 (por defecto).
3
Incluye: pgAdmin (interfaz gráfica) + psql (línea de comandos).
4
Verificar: psql --version
🔁 Git — Control de Versiones
Instalación y configuración
1
Descargar desde https://git-scm.com/downloads
2
Instalar → configuración inicial:
✓
git config --global user.name "Tu nombre"
git config --global user.email "tu@email.com"
3
Verificar: git --version
